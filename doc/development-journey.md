# 民大课表 App 开发历程（又名：一个人的踩坑日记）

## 起因

事情很简单：**超级课程表导不了我们学校的课**。

大连民族大学升级了新版教务系统，超级课程表完全识别不了。每个选修课都需要手动导课表，终于忍不了了——自己写一个。

## 第一关：搞清楚教务系统到底长啥样

本以为教务系统就是个普通网页，打开 Chrome DevTools 一看——好家伙，**三层 iframe 套娃**：

```
主页 xsMainV.htmlx
  └─ person iframe (xsMainV_new.htmlx)
       └─ xskb_list.do
            └─ #Iframe1 (xskb_list.do?viweType=0)
```

而且还有**两种完全不同的 HTML 结构**，取决于你从哪个入口进去：
- 一种是课程信息塞在一个 `<span>` 里用分号分隔（老实说谁设计的）
- 另一种是每个字段单独一个 `<div>`

两种都得兼容，不然总有人导不了。

## 第二关：时间段的坑

学校的课表时间看起来简单，实际上：
- 周次可以是 `10-13,17-18周` 这种**逗号分隔多段**格式
- 跨节课用 `rowspan` 合并行，导致后续行的列号会**错位**
- 一开始我还多配了一个 `12:10-12:55` 的时间段——这个时间段根本不存在，是午休！下午课全部往上错了一格，排查了好一会儿

## 第三关：切周闪烁——最折磨的 Bug

为了实现"像切手机桌面一样左右滑动切周"，用了三页 ScrollView + pagingEnabled 的方案。效果确实丝滑，但带来了一个令人抓狂的 Bug：

> 从第8周滑到第9周时，会**闪一下第10周的课**。

原因：滑到第3页后触发切周，React 重新渲染三页数据 `[8,9,10]`，但 ScrollView 还停在第3页（现在是第10周），要等 `scrollTo` 回到中间页才显示第9周。**数据更新和滚动位置重置无法在同一帧完成**。

尝试过的修复方案：
1. `useEffect` + `scrollTo` → 还是闪
2. `useLayoutEffect` → React Native 的 scrollTo 不是真同步，还是闪
3. 先 scrollTo 再改数据 → 闪的变成上一周了
4. 单页动画方案 → 滑动手感变差，用户不满意
5. **白色遮罩方案** → 切换时盖一层白，数据就绪后移除。视觉上从"闪错误内容"变成"短暂白屏"，勉强能接受

最终用的是方案5。不完美，但至少不会看到错误的课了。

## 第四关：Expo Go 版本不兼容

装好 Expo Go，扫码一连——

> "Project is incompatible with this version of Expo Go"

Expo SDK 55 太新，Google Play 上的 Expo Go 还没跟上。于是放弃 Expo Go，改用 EAS 云端构建 APK。

## 第五关：EAS 构建额度用完

EAS 免费版每月 30 次构建。一个下午调 Bug 调得太猛，**额度用完了**。下次恢复要等到下个月。

于是又装了 Android Studio（3-4GB），配环境变量，配 JAVA_HOME，第一次 Gradle 构建下载依赖等了 20 分钟……

## 第六关：登录问题——倒霉催的

为了支持"切换账号"功能，给 WebView 加了 `incognito={true}`——结果教务系统登录直接报**"用户名/密码不能为空"**。

去掉 incognito，加 `cacheEnabled={false}` → 还是空。
去掉 cacheEnabled → 还是空。
去掉所有额外属性 → 还是空。
恢复到最简 WebView → **还是空**。

折腾了一个多小时，最后发现：

> **是学校教务系统自己挂了。** 😐

Chrome 浏览器直接打开也登不了。不是我的锅。

## 第七关：设置弹窗的手势冲突

设置页面有个日期滚轮选择器（WheelPicker），外面包了个 ScrollView。结果：
- 外层 ScrollView 吃掉了滚轮的滑动手势
- `onStartShouldSetResponder` 又把点击事件拦截了
- 滚轮必须先点一下输入框才能滑动

最后改成分层架构：背景层单独处理关闭，内容区独立，滚轮才恢复正常。

## 现状

App 核心功能已经跑通：
- ✅ 从教务系统一键导课
- ✅ 周课表视图 + 左右滑动切周
- ✅ 手动加课
- ✅ 课程详情弹窗
- ✅ 本地存储离线查看
- ⚠️ 切换账号功能暂不可用（WebView session 问题）
- ⚠️ 切周时偶尔有短暂白屏（架构限制）

## 感想

一个"看起来很简单"的课表 App，实际开发中遇到的坑比想象中多得多。三层 iframe 套娃、两种 HTML 结构、ScrollView 手势冲突、React Native 动画时序问题、Expo 版本兼容、构建额度限制……甚至还赶上了学校官网宕机。

但至少——**我再也不用手动导课表了**。
